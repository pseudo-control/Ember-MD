‚ĚĮ Same principle as SnapDraw but actually easier in one way. NGL Viewer has a   
rich JavaScript API on the stage object. So even though the rendering is WebGL  
(which is even more opaque than Konva's 2D canvas ‚ÄĒ Playwright can't even do    
meaningful screenshot diffing on WebGL because of GPU nondeterminism), the      
internal state is fully queryable.                                              
                                                                                
If you expose the NGL stage:                                                    
                                                                                
```js                                                                           
if (import.meta.env.MODE === 'test') window.__nglStage = stage;                 
```                                                                             
                                                                                
Then Claude can write assertions like:                                          
                                                                                
```js                                                                           
// Verify structure loaded

  () => window.__nglStage.compList.length

  () => window.__nglStage.compList[0].reprList.length                           
);
```                                                                             

For your interactive functions ‚ÄĒ distance measurement, residue highlighting,    
pocket surface toggling, whatever you're adding ‚ÄĒ each one presumably modifies  
NGL state (adds a representation, changes visibility, updates a selection).     
Claude can fire the mouse/keyboard events through Playwright and then verify    
the state change through the API. The loop handles iterating through each       
interactive feature.                                                            
                                                                                
The one thing that's genuinely hard is testing visual correctness ‚ÄĒ "does this  
surface rendering look right" or "is the ligand colored correctly." For that    
you'd need screenshot baselines that you approve manually, and even then WebGL  
rendering varies across GPUs so the diffs can be flaky. But functional          
correctness (did the interaction change the right state) is fully testable this 
 way.

when you're doing normal interactive development.
